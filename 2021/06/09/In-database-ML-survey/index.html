In-Database ML

In-Database Machine Learning 调研

背景

数据库与机器学习结合：

  • 更简单。熟悉数据库的用户可快速上手，无需额外学习成本。
  • 更高效。在存储数据的地方直接使用机器学习，无需迁移数据。
  • 更直观。直接通过SQL获取预测结果，无需复杂的模型部署和管理。

对于第二点，不是所有系统都满足。

现状

现有的工业产品

  • 闭源：
    • Amazon Redshift ML
    • Google Cloud BigQuery ML
    • Microsoft SQL Server
    • …
  • 开源:
    • MindsDB
    • Apache MADlib
    • SQLFlow
    • …

闭源的只能通过文档观察产品形态，推测思路

开源的根据需求后续跟进分析

MindsDB

MindsDB is an open-source AI layer for existing databases that allows you to effortlessly develop, train and deploy state-of-the-art machine learning models using SQL queries.

mindsdb/mindsdb: Predictive AI layer for existing databases. (github.com)

Apache MADlib

https://github.com/apache/madlib

对比

是否提供SQL command 数据库支持 算法支持 实现方式
Amazon Redshift ML ✔ 内部 ML, DL 训练和推理解耦
Google Cloud BigQuery ML ✔ 内部 ML, DL 解耦❓
Microsoft SQL Server ✔ 内部 ML, DL 解耦❓
MindsDB ✔ 较多且可拓展 ML,（正在做DL） 解耦
Apache MADlib ✔ 可拓展 ML,（正在做DL） 耦合

实现方式想要描述的是：一般推理可以到数据库内部来做（相对容易），但模型训练是否在数据库内部来做？目前只看到MADlib在数据库内部实现训练算子，模型的训练和推理都可以在数据库内完成，这种方式实现难度比较大。
